# Microbenchmark Evaluation

## Overview

This folder contains scripts to run benchamrks-tests of page deduplication on:

- Memory savings
- Read latency
- Write latency

It also generates plots automatically inside a `plots/` directory in the current folder.

## Requirements

Install the required system tools and Python packages:
```
    sudo apt update
    sudo apt install python3 fio
    sudo apt install python3-pandas
    sudo apt install python3-matplotlib
```
## Running the Benchmarks
We disable swap memory for safely running of these tc. this is just a precautionary measure as these tc will noto trigger swap memory.
To run all benchmarks and generate plots:
```
    sudo swap_off -a
    sudo ./run_all.sh
```
This will run the memory, read, and write benchmarks, and save the generated plots in the `plots/` folder.

## Benchmark Methodology

### Memory Benchmarks

The memory benchmark was performed using both single-file and multi-file test cases. For each case, different percentages of identical pages were generated to simulate varying levels of sharing. The memory usage or cache size was measured before and after deduplication, and the savings were recorded for each sharing percentage.

### Read Benchmarks

The read benchmark measured the effect of deduplication on read latency. Files of different sizes were tested with varying shared-page percentages. Since read operations will access the same address inside the cpu for shared pages, they may get cpu-cache hits, so an improved read latencywas noticed with higher %shared pages.

### Write Benchmarks

The write benchmark measured the impact of deduplication on write latency. The same file sizes and sharing patterns were used as in the read tests. Since writes to shared pages trigger copy-on-write, the benchmark captures the additional latency introduced when a shared page must be duplicated before being modified.

## FIO Configuration

The following FIO configuration was used for random read and write testing:

    fio --name=randread \
        --filename="$file" \
        --rw=randread/write \
        --bs=4k \
        --size="${size}M" \
        --ioengine=sync \
        --direct=0 \
        --numjobs=8 \
        --group_reporting \
        > /dev/null 2>&1

### Parameter Details

- `--rw=randread/write` : random read or write workload
- `--bs=4k` : 4 KB block size
- `--size="${size}M"` : test file size in MB
- `--ioengine=sync` : blocking read, writes
- `--direct=0` : uses page cache
- `--numjobs=8` : runs 8 parallel jobs
- `--group_reporting` : combines job output into a single report

## Results Summary

- Memory savings increase as the percentage of shared pages increases.
- Read latency improved for higher %pages shared file on deduplication.
- Write latency increases when more shared pages are present, due to copy-on-write overhead.

## Notes

- Run the benchmarks in a controlled environment with minimal background activity.
- Make sure there is enough RAM available to avoid swapping.
- The plots generated by the scripts summarize the behavior across all test cases.
